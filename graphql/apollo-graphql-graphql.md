# Apollo GraphQL GraphQL API

Whether your team is new to GraphQL, or seasoned experts, learn why Apollo is the fastest and safest way to build and scale your APIs.

## Apollo GraphOS Platform API

The GraphOS Platform API enables programmatic management of Apollo GraphOS, including publishing schemas, running schema checks, managing proposals, handling API keys, and accessing operation and field insights data.

**Documentation:** https://www.apollographql.com/docs/graphos/platform/platform-api

**References:**

- Documentation: https://www.apollographql.com/docs/graphos/platform/platform-api
- GettingStarted: https://www.apollographql.com/docs/graphos/platform

## Apollo Server

Apollo Server is an open-source, spec-compliant GraphQL server compatible with any GraphQL client. It is the recommended way to build a production-ready, self-documenting GraphQL API that can use data from any source.

**Documentation:** https://www.apollographql.com/docs/apollo-server

**References:**

- Documentation: https://www.apollographql.com/docs/apollo-server
- APIReference: https://www.apollographql.com/docs/apollo-server/api/apollo-server

## Apollo Client (Web)

Apollo Client is a comprehensive state management library for JavaScript and TypeScript applications. It is the industry-leading GraphQL client for React, Vue, Angular, and more, with powerful caching, intuitive APIs, and comprehensive developer tools.

**Documentation:** https://www.apollographql.com/docs/react

**References:**

- Documentation: https://www.apollographql.com/docs/react
- GettingStarted: https://www.apollographql.com/docs/react/get-started
- APIReference: https://www.apollographql.com/docs/react/api/core/ApolloClient

## Apollo Client (iOS)

Apollo iOS is a Swift-first, open-source GraphQL client for native iOS applications. It executes queries and mutations using a GraphQL server and returns results as operation-specific Swift types, with built-in caching mechanisms for GraphQL data.

**Documentation:** https://www.apollographql.com/docs/ios

**References:**

- Documentation: https://www.apollographql.com/docs/ios

## Apollo Client (Kotlin)

Apollo Kotlin is a strongly-typed, caching GraphQL client for the JVM, Android, and Kotlin multiplatform. It generates Kotlin models for GraphQL operations and executes them against a GraphQL server, returning results as operation-specific Kotlin types.

**Documentation:** https://www.apollographql.com/docs/kotlin

**References:**

- Documentation: https://www.apollographql.com/docs/kotlin

## Apollo Router

Apollo Router is a high-performance graph router written in Rust for running federated supergraphs using Apollo Federation 2. It serves as the execution engine for graph-based API orchestration, sitting in front of existing REST and GraphQL APIs.

**Documentation:** https://www.apollographql.com/docs/graphos/routing

**References:**

- Documentation: https://www.apollographql.com/docs/graphos/routing

## Apollo Connectors

Apollo Connectors simplify REST API orchestration in graphs, replacing unmaintainable procedural code with scalable, declarative schema directives. They provide a declarative language for connecting REST APIs directly to a graph without writing a separate GraphQL server.

**Documentation:** https://www.apollographql.com/docs/graphos/connectors

**References:**

- Documentation: https://www.apollographql.com/docs/graphos/connectors
- GettingStarted: https://www.apollographql.com/docs/graphos/connectors/getting-started

## Rover CLI

Rover is the command-line interface for managing and maintaining graphs with Apollo GraphOS. It supports publishing subgraph schemas, running a supergraph router locally, fetching schemas via introspection, composing federated supergraph schemas, and running schema checks.

**Documentation:** https://www.apollographql.com/docs/rover

**References:**

- Documentation: https://www.apollographql.com/docs/rover
- GettingStarted: https://www.apollographql.com/docs/rover/getting-started

## Apollo Federation

Apollo Federation enables declarative composition of multiple APIs into a single federated GraphQL API. It serves as an API orchestration layer where clients make a single GraphQL request and the router coordinates multiple API calls to return a unified response.

**Documentation:** https://www.apollographql.com/docs/graphos/schema-design/federated-schemas/federation

**References:**

- Documentation: https://www.apollographql.com/docs/graphos/schema-design/federated-schemas/federation

## Apollo MCP Server

Apollo MCP Server provides Model Context Protocol integration for Apollo GraphQL, enabling AI agents and large language models to interact with GraphQL APIs through the MCP standard.

**Documentation:** https://www.apollographql.com/docs/apollo-mcp-server

**References:**

- Documentation: https://www.apollographql.com/docs/apollo-mcp-server
